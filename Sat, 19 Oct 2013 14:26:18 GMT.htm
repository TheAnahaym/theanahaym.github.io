• Создание виртуальной машины
    • Hyper-V 2012 R2
    • Экспорт и импорт ВМ

 

 Создание и настройка новой виртуальной машины немного отличается от предыдущего описания для Windows Server 2012.

 Если сразу нажать Finish, то будет создана ВМ первого поколения.

Выбираем название нашей ВМ. Если нужно, можем выбрать место хранения конфигурационных файлов.

Выбираем поколение ВМ: первое или второе. Собственно, здесь и написано какие гостевые ОС поддерживаются для второго поколения ВМ. С Unix-like пока не тестировал. После выбора поколения ВМ его нельзя будет изменить.

Выделяем объём для оперативной памяти для ВМ, в соответствии с требованиями к устанавливаемой ОС.

Выбираем тип сетевой карты. У ВМ второго поколения нет Legacy Network Adapter.

Создаём жёсткий диск и выбираем его размер. Можно использовать уже имеющийся файл с образом системы. Не wim, а vhdx.

 Можно подключить установочный диск, или выполнить это позже. Установка не может производится с хостового оптического привода. Если выбрать "установка ОС позже", то в созданной ВМ будет отсутствовать DVD-ROM и загрузка по умолчанию будет через сеть.

 Далее заходим в свойства новой ВМ и добавляем процессорной мощности в зависимости от выполняемых ролей на ВМ.

 При попытке установки Windows Server 2008 R2 мы увидим следующее ниже сообщение. Как интересно, Microsoft Hyper-V  не доверяет Microsoft OS. Отключение Secure Boot не поможет.
