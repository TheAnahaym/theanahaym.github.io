 Перенаправляемые папки включаются в групповой политике. Официальная документация на английском языке для Server 2003. Для Server 2012 практически ничего не изменилось. Основные изменения связанны с тем, что структура папок у 2000\ХР и Vista\7\8 различная. Создаём политику для перенаправления Рабочего стола, Моих Документов и Данных приложений. Я рекомендую разделять перенаправляемые папки в разных политиках, т.к. это позволяет больше гибкости в дальнейшей эксплуатации перенаправляемых папок. Например, у одного пользователя было обнаружено, что его приложение (3Dmax) постоянно аварийно завершает работу. После отключения политики перенаправления AppData для этого пользователя авария ушла. Так же данная политика была отключена веб-программистам, т.к. FireFox сильно терял в производительности при включённой политике. Если бы все пользовательские папки были в одной политике, то отключить только AppData было бы невозможно. Политики применялись на основе фильтра по группам безопасности, которые должны быть созданы и содержать нужных вам пользователей.

 Есть несколько вариантов перенаправления папок. Я рассмотрю самый распространённый вариант: папка Рабочего стола создаётся на сервере для каждого пользователя.

 В настройках политики необходимо снять галку с поля "назначить пользователю эксклюзивные права на Рабочий стол". С включённой опцией администратор не будет иметь доступа к папке на сервере. При этом, если папка уже была создана то отключение опции не назначит права администратору. Нужно создовать новую политику или удалять (перемещать) данные пользователя.

 Второй важной настройкой является поведение перенаправялемых папок в случае, если политика отключена. По умолчанию вам предлается выбрать новое расположение общего ресурса. Я же выбираю - Переместить папку обратно в локальный профиль пользователя, который может быть перемещаемый. Мой выбор связан с простотой в управление перенаправляемыми папками в случае замены файлового хранилища. К сожалению, часто не получалось переместить сразу на новое хранилище, и тогда приходилось руками перемещать данные, да ещё и правильно назначать права. Я думаю, ничего страшного не произойдёт, если данные скопируются на локальный компьютер. Если конечно там уже не террабайты данных... Но надо с опцией перемещения в новое расположение разобраться.

 А вот что видит пользователь, когда создаёт папку на рабочем столе.

 Если одновременно включить перемещаемый профиль и перенаправление папки рабочего стола для нового пользователя, то на сервере создатся папка с профилем без папки рабочего стола. Если включить политику перенаправления папок для пользователя, у которого уже есть перемещаемый профиль, то папка рабочего стола останется в его профиле. Но её можно удалить после, конечно, резервной копии )
