В данной статье будет рассмотрена установка Exchange 2016 в автоматическом режиме на контроллер домена под управлением Windows 2012 R2. Установка на отдельный сервер ничем не отличается. Предварительные требования к установке Exchange 2016:
- программные и аппаратные требования для Exchange 2016
- требования к компонентам для Exchange 2016
- Microsoft .NET Framework 4.5.2
- Unified Communications Managed API 4.0 Runtime (API 5.0)
- права Enterprise Admin или делегирование прав при наличии Exchange 2013
- сервер установки:

 Изначально статья планировалась о нормальной установке, но там практически ничего не изменилось, по этому в целях экономии времени будем устанавливать атоматически. Все варианты автоматической установки описаны в этой статье.
Установка необходимых компонентов для ролей Mailbox:

Install-WindowsFeature AS-HTTP-Activation, Desktop-Experience, NET-Framework-45-Features, RPC-over-HTTP-proxy, RSAT-Clustering, RSAT-Clustering-CmdInterface, RSAT-Clustering-Mgmt, RSAT-Clustering-PowerShell, Web-Mgmt-Console, WAS-Process-Model, Web-Asp-Net45, Web-Basic-Auth, Web-Client-Auth, Web-Digest-Auth, Web-Dir-Browsing, Web-Dyn-Compression, Web-Http-Errors, Web-Http-Logging, Web-Http-Redirect, Web-Http-Tracing, Web-ISAPI-Ext, Web-ISAPI-Filter, Web-Lgcy-Mgmt-Console, Web-Metabase, Web-Mgmt-Console, Web-Mgmt-Service, Web-Net-Ext45, Web-Request-Monitor, Web-Server, Web-Stat-Compression, Web-Static-Content, Web-Windows-Auth, Web-WMI, Windows-Identity-Foundation

 Никакая подготовка леса\домена не требуется, т.к. это всё сделает установщик и в графическом интерфейсе:

И во время автоматической установки из командной строки:

Setup.exe /mode:Install /role:Mailbox /OrganizationName:MyOrg /IAcceptExchangeServerLicenseTerms

для установки на другой жёсткий диск используйте ключ /TargetDir:"D:\Exchange Server"
и ключ для расположения логов на другой диск: /LogFolderPath:"E:\Logs"

После перезагрузки сервера установка Exchange 2016 будет завершена.
