    • Создание виртуальной машины
    • Экспорт и импорт ВМ

 

 
 Новая операционная система Windows Server 2012 R2 явила нам новую верисию гипервизор Hyper-V, в котором появились значительные улучшения, но, к сожалению, после беглого взляда по возможностям Hyper-V остался и неприятный осадок. Это связано с тем, что некоторые новшества накладывают органичения на операционные системы используемые как гостевые. Но обо всё по порядку.

 Панель управления Hyper-V по сравнению с предыдущей версией в Windows Server 2012 практически не изменилась. Основные изменения начинаются во время создания виртуальной машины. По этому для панели управления я просто повторю описание прошлой статьи. Snapshot теперь называется Checkpoint.

 - New - меню создания виртуальный машины, диска или флоппи диска.
 - Import Virtual Machine - импорт виртуальной машины (ВМ). Можно выполнять без экспорта с другого сервера.
 - Hyper-V Settings - настройки сервера по умолчанию: где хранить образы дисков, настройки ВМ, поведение клавиатуры и мыши и др.
 - Virtual Switch Manager - управление виртуальными сетями.
 - Virtual SAN Manager - управление дисками SAN
 - Edit disk - редактирование существующими образами дисков.
 - Inspect disk - просмотр информации о диске.
 - Stop Service - остановка службы Hyper-V
 - Remove server - удаление сервера

 Далее идёт меню для выбранной работающей ВМ.

 - Connect - подключение через Virtual Machine Connection. Анналог подключения через консоль.
 - Settings - настройки ВМ
 - Start, Turn Off, Shut down, Save, Reset - запуск, выключение по питанию, выключение, сохранение состояния и сброс ОС. Выключение поддерживается только ОС Windows.
 - Checkpoint - снимок ВМ. Сохранение состояния рабочей системы, к которой можно откатится в случае неполадок.
 - Move - перемещение ВМ или образа диска.
 - Export - экспорт ВМ.
 - Rename - переименование и удаление ВМ.
 - Enable Replication - разрешение репликации между нодами кластера Hyper-V.

 Теперь рассмотрим базовые настройки (необходимые для функционирования ВМ)  виртуальной машины второго поколения. Виртуальные машины второго поколения - глобальное изменение в Hyper-V 2012 R2. А вот и неприятное ограничение для ВМ 2.0 - гостями могут быть только операционные системы Windows 8, Server 2012 и выше. Т.е. установить Windows 7 или Windows Server 2008 R2 и младше на UEFI не получится. BIOS с поддеркой UEFI имеет множество преимуществ перед обычным BIOS без поддержки UEFI. Одно из - загрузка с GPT дисков, которые не имеют ограничения в 2 TB, как MBR диски.

 

 В виртуальной машине второго поколения отсутствуют IDE контроллеры, COM порты и дискеты. Так же по умолчанию в свежесозданной ВМ отсутствует DVD-ROM. В настройках можно отключить Secure Boot.

Организация сети в Hyper-V не изменилась. Всё так же есть выбор из трёх типов сети. Благодаря новому функционалу в службе RRAS для ВМ можно использовать интеграцию с мультитенатным шлюзом в развернутой ВМ. Повторение описание типов сетей:

 - Изолированная сеть Private имеет доступ только в пределах своей сети. ВМ с изолированной сетью взаимодействует только с ВМ в той же сети. При этом разные изолированные сети так же не могут между собой взаимодействовать. Однако, между двумя ВМ в разных сетях можно установить третий сервер с двумя сетевыми интерфейсами из каждой сети, и настроить RRAS как Lan routing. Для взаимодействия с интернетом RRAS должен быть настроен как NAT (вторая сетевая карта должна быть типа Внешняя).

 - Внутренняя сеть Internal может может взаимодействовать с физической сетевой картой сервера Hyper-V. При этом на самом хосте создаётся виртуальный интерфейс, который можно\нужно настроить, если на ВМ не установлен DHCP сервер. Можно на хосте установить роль RRAS. Тогда ВМ можно подключить к интернету через NAT. Данный тип подключения - отличный способ протестировать различные программные шлюзы, но установленные на хосте. Позволяет возможность доступа по RDP к Windows ВМ и по SSH к Unix-like ВМ. Будьте внимательны: установка роли RRAS снимает бесплатную лицензию на хост!

- Внешняя сеть External использует одну из (если их несколько) сетевых карт хостового сервера. Напрямую взаимодействует с внешними ресурсами. К примеру, ВМ может получить IP адрес от вашего роутера или модема, если там включена служба DHCP, а следовательно, будет находится в одной локальной сети с самим сервером Hyper-V. Если же службы DHCP в сети нет, то сетевая карта ВМ вручную настраивается так же, как и сетевая карта хоста. Естественно, что IP адреса не могут быть одинаковыми. Позволяет возможность доступа по RDP к Windows ВМ или по SSH к Unix-like ВМ, а так же прямому доступу ВМ к интернету через установленный маршрутизатор или модем.
